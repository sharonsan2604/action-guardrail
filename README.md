# Action Guardrail (SentinelAI) — PS-3.1

SentinelAI is an AI safety and governance guardrail system designed to intercept, audit, and regulate structured actions (tool/function calls) generated by AI Agents before execution. 

Unlike traditional AI safety systems that only filter textual output (what the LLM *says*), SentinelAI filters what the agent actually *does* (e.g. database deletes, email dispatches, file reads), providing a safety gate with three key outcomes: **Block**, **Hold for Human Review (HITL)**, or **Log + Allow**.

---

## 📖 Problem Statement
When an AI agent decides to take an action, it might produce an instruction like:
`delete_records(table="customers", count=50000)`
Traditional content filters pass this because it lacks toxic language. However, executing this would delete critical data. SentinelAI bridges this safety gap by introducing an active interception layer.

## 🏛️ Architecture & Request Flow

```
                   +----------------------------------+
                   |          User / Browser          |
                   +----------------------------------+
                                    |
                                    v
                   +----------------------------------+
                   |    React Web Dashboard (8501)    |
                   +----------------------------------+
                                    |
                                    v
                   +----------------------------------+
                   |      FastAPI Server (8000)       |
                   +----------------------------------+
                                    |
            +-----------------------+-----------------------+
            |                                               |
            v                                               v
  [ AI Agent (agent.py) ]                         [ Core Guardrail Engine ]
            |                                               |
            +------------> [ Gemini LLM ]                   +---> [ rules.yaml ]
            |              (Free-Tier Online)               |
            |                                               v
            +------------> [ Local Pattern Parser ]      Evaluate Conditions
                           (Fallback Offline)               |
                                                            |
                     +-------------------+------------------+
                     |                   |                  |
                     v                   v                  v
                 [ ALLOW ]           [ BLOCK ]           [ HITL ]
                     |                   |                  |
                     v                   |                  v
               [ tools.py ]              |           [ Review Queue ]
            (Execute Tool Call)          |                  |
                     |                   |                  v
                     +-------------------+------------> Approve/Reject
                                                            |
                                                            v
                                                   [ SQLite Audit Log DB ]
```

### Flow of a Request
1. **Instruction input**: A user submits a natural language instruction (e.g., *"Delete 500 records"*).
2. **Action resolution**: The `Agent` parses the message. It attempts **Google Gemini API** (using the free-tier `gemini-1.5-flash` model), but falls back automatically to the **Local Pattern Parser (Offline)** if no API Key is configured.
3. **Guardrail evaluation**: The FastAPI server routes the resolved tool call to `Guardrail` which runs condition matching from `rules.yaml` (e.g., `params.count > 100`).
4. **Execution Decision**:
   - **Allowed**: The mock tool in `tools.py` executes.
   - **Blocked**: Action is rejected instantly.
   - **Pending Review**: Held in the HITL (Human-in-the-Loop) database queue for human approval.
5. **Auditing**: All events, reviews, and execution details are logged securely into `guardrail.db`.

## 💻 Tech Stack
* **Language**: Python 3.10+
* **Framework**: FastAPI (REST Web Services)
* **LLM Engine**: Google Gemini API (`gemini-1.5-flash` Free-Tier) & Offline Pattern Heuristics
* **Database**: SQLite (Storage) & SQLAlchemy (ORM)
* **Frontend**: React + Vite (Dashboard) & Streamlit (Console Dashboard fallback)
* **Containerization**: Docker & Docker Compose
* **Unit Tests**: Pytest

## ⚙️ Environment Variables
The configuration variables are loaded from the `.env` file in the project root:

| Key | Description | Example Value |
| :--- | :--- | :--- |
| `GEMINI_API_KEY` | Optional Google AI Studio Key. Fallback to Local Offline Parser if missing. | `AIzaSy...` |
| `COMPANY_DOMAIN` | Internal company domain filter. Outbound emails outside this are held for review. | `mycompany.com` |
| `DATABASE_URL` | SQLAlchemy database storage path. | `sqlite:///./guardrail.db` |
| `DRY_RUN` | Global dry-run toggle. If true, policies are assessed but no tool calls execute. | `false` |

## 🚀 Quick Start (Local Setup)

### Prerequisites
* Python 3.10 or higher
* Node.js (only for local manual frontend compilation)

### 1. Configure Environment
Clone/open the workspace directory and create your `.env` configuration file:
```bash
cp .env.example .env
```
*(Optionally paste your free Gemini API Key under `GEMINI_API_KEY` to test online AI agent integrations. Leave it empty to run using the offline regex fallback).*

### 2. Activate Virtual Environment & Install Dependencies
```bash
# Windows PowerShell
.\venv\Scripts\Activate.ps1

# Install requirements
pip install -r requirements.txt
```

### 3. Run Simulation Harness
Execute the console script to check the core guardrail evaluations:
```bash
python main.py
```

### 4. Start the Backend API
Start FastAPI server with hot-reload:
```bash
uvicorn api:app --reload --port 8000
```
Open [http://localhost:8000/docs](http://localhost:8000/docs) in your browser to view the interactive API documentation.

### 5. Start the React Frontend Dashboard
```bash
cd frontend
npm install
npm run dev
```
Open [http://localhost:8501](http://localhost:8501) to interact with the SentinelAI dashboard.

## 📍 API Endpoints

### Health & Configuration
* `GET /health`: Returns system status and API version.
* `GET /rules`: Returns the configurations loaded from `rules.yaml`.

### Decision Pipeline
* `POST /act`: Directly evaluates and executes a JSON action block.
* `POST /request`: Main agent pipeline. Evaluates raw English requests via Gemini/Offline parser and applies guardrail.

### Audit & Human Review (HITL)
* `GET /audit`: Returns recent audit trail logs (filterable by outcome).
* `GET /audit/{entry_id}`: Retrieves specific audit record details.
* `GET /review`: List of held items awaiting human approval.
* `POST /review/{entry_id}/approve`: Approves a pending review request and executes the tool.
* `POST /review/{entry_id}/reject`: Rejects a pending review and updates logs.
